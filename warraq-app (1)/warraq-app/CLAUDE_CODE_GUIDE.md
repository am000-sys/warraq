# 🚀 دليل البدء السريع لـ Claude Code

> هذا الملفّ مكتوب لمستخدم المشروع. إذا كنتَ Claude Code، اقرأ `CLAUDE.md` بدلاً من هذا.

---

## ما هو Claude Code؟

أداة من Anthropic لمساعدتك على البرمجة من سطر الأوامر (Terminal). لا تحتاج أن تكون مبرمجاً محترفاً.

---

## التثبيت (مرّة واحدة)

### الخطوة ١: ثبّت Claude Code
افتح Terminal وشغّل:
```bash
npm install -g @anthropic-ai/claude-code
```

### الخطوة ٢: سجّل الدخول
```bash
claude
```
سيطلب منك تسجيل الدخول بحساب Claude (نفس حسابك على claude.ai).

---

## استخدام Claude Code مع وَرَّاق

### الخطوة ١: افتح المشروع
```bash
cd warraq-app
claude
```

### الخطوة ٢: ابدأ بهذا الأمر الأوّل
انسخ والصق هذا في Claude Code:

```
اقرأ ملفّ CLAUDE.md في جذر المشروع. هذا دليلك المرجعيّ.
بعدها، اقرأ design-reference/warraq-v3.html بالكامل لتفهم التصميم النهائي
الذي صنعتُه.

ثمّ ابدأ بالمرحلة ١ من خطّة العمل (في CLAUDE.md):
استبدل الواجهات الحاليّة بالتصميم من v3.html، بدءاً من المكوّنات
المشتركة، ثمّ الصفحة الرئيسية، ثمّ صفحات المصادقة.

اعمل صفحة واحدة في كلّ مرّة. أرني كلّ صفحة بعد إكمالها قبل الانتقال
للتي تليها.
```

### الخطوة ٣: شغّل الموقع للمعاينة
في **Terminal آخر** (نافذة جديدة):
```bash
cd warraq-app
npm run dev
```
افتح `http://localhost:3000` في المتصفّح.

كلّما يُعدّل Claude Code ملفّاً، الموقع يتحدّث تلقائياً.

## أوامر مفيدة في Claude Code

### استمرار جلسة سابقة
```bash
claude --continue
```

### رؤية كلّ الملفّات
في Claude Code:
```
أرني هيكل المشروع كاملاً
```

### إصلاح مشكلة محدّدة
```
الصفحة الرئيسية لا تظهر بشكل صحيح. افتحها في المتصفّح
وافحص ما المشكلة.
```

### معاينة قبل التطبيق
دائماً اطلب من Claude Code:
```
أرني التغييرات قبل تطبيقها
```

## ترتيب العمل المقترح

اطلب من Claude Code أن ينفّذ بهذا الترتيب:

### الجلسة الأولى (٣٠-٤٥ دقيقة)
1. قراءة CLAUDE.md و v3.html
2. إنشاء مكوّنات مشتركة (Nav, Footer — بعضها جاهز)
3. الصفحة الرئيسية (موجودة بشكل أوّلي - تحسينها)
4. صفحة الأسعار

### الجلسة الثانية (٤٥ دقيقة)
1. صفحات المصادقة (login, signup, forgot-password, reset-password)
2. لوحة المستخدم (layout + dashboard)

### الجلسة الثالثة (٤٥ دقيقة)
1. صفحات الرفع والوظائف
2. صفحة الفوترة والمدفوعات
3. صفحات الإعدادات و API keys

### الجلسة الرابعة (٣٠ دقيقة)
1. صفحات المؤسسات
2. لوحة المالك
3. اختبار شامل

### الجلسة الخامسة (متغيّر)
1. حلّ معالجة PDF (انظر CLAUDE.md القسم ٧)
2. تجربة كلّ ميزة

## أمثلة على طلبات جيّدة

✅ **محدّد وواضح:**
```
في src/app/(app)/dashboard/page.tsx، طبّق التصميم من function Dashboard
في design-reference/warraq-v3.html. حافظ على المنطق الموجود لجلب البيانات
من قاعدة البيانات.
```

✅ **عملي:**
```
الصفحة /upload لا تَعمل عند رفع ملفّ. افتحها في المتصفّح، جرّب رفع ملفّ،
وأخبرني ما الخطأ في console.
```

✅ **خطوة بخطوة:**
```
قبل أن تنتقل للصفحة التالية، شغّل npm run dev وتأكّد من أنّ صفحتي
الرئيسية والأسعار تَعملان بدون أخطاء.
```

❌ **مبهم:**
```
اعمل كلّ شيء.
```

❌ **خارج النطاق:**
```
اِبنِ تطبيق iOS.
```

## مشاكل شائعة وحلولها

### "Module not found"
```
شغّل npm install
```

### "Database error"
```
شغّل: npx prisma db push && npx prisma generate
```

### "Hot reload لا يَعمل"
- أوقف `npm run dev` (Ctrl+C)
- شغّله مرّة أخرى

### Claude Code "نسي" السياق
```bash
claude --continue
```
أو ذكّره:
```
اقرأ CLAUDE.md من جديد
```

## نصائح أخيرة

1. **اقرأ التغييرات قبل قبولها** — Claude Code يعرض لك diff
2. **استخدم Git** بعد كلّ مرحلة:
   ```bash
   git add .
   git commit -m "إكمال صفحات Marketing"
   ```
3. **لا تتعجّل** — صفحة واحدة في كلّ مرّة أفضل من كلّ شيء دفعة واحدة
4. **اسأل** عند الحاجة:
   ```
   لماذا اخترتَ هذا الحلّ بدلاً من الآخر؟
   ```
